# 用户使用指南

本工具通过 8 个步骤，引导你完成一套完整的 Delphi-AHP 学术研究流程。

---

## 准备工作

### 1. 获取 API Key

本工具需要调用 LLM（大型语言模型）来模拟专家访谈。你需要先获取一个 API Key。

**推荐使用 Xi-API**（聚合平台，一个账号使用多种模型）：

1. 打开注册链接：[点击注册 Xi-API](https://api.xi-ai.cn/register?aff_code=GXZx)
2. 注册账号并登录
3. 进入个人中心 → API Key → 创建新的 API Key（复制保存好）
4. 注意查看每个模型的费率，选择适合学生党的便宜模型

其他可选平台：DeepSeek、Moonshot、Kimi 等均可，但需要分别在各自官网注册。

### 2. 启动程序

- **Windows**：双击 `run.bat`
- **macOS**：终端运行 `bash run.sh`

启动后会看到主菜单，按方向键↑↓选择菜单项，空格选中，回车确认。

## 步骤详解

---

### 步骤 1/8：项目设置

这一步是输入你的研究信息。

**需要填写的内容：**

- **研究标题**：你的论文研究主题，例如"直播电商主播信任机制对消费者复购意愿的影响研究"
- **分析框架**：你使用的分析方法，例如 PEST、SWOT、Porter's Five Forces 等
- **研究背景**：为什么做这个研究，研究的重要性和意义
- **研究目的**：通过研究想达到什么目标
- **研究边界**：研究对象的范围和限制条件
- **研究问题**：你想通过专家访谈回答的核心问题（至少一个）
- **访谈语言**：选择专家访谈使用的语言（中文或English）

填写完成后，会显示确认菜单：
- 选择 **1** 确认并继续
- 选择 **2** 修改某个内容

---

### 步骤 2/8：API 配置

这一步是配置 LLM API，让程序能够调用 AI 进行专家访谈。

**操作步骤：**

1. 选择 **添加 API**（方向键选中后回车）
2. 输入 API Key（粘贴你从 Xi-API 或其他平台复制的 Key）
3. 选择模型（推荐选择便宜且效果好的模型，如 DeepSeek 或 Moonshot）
4. 选择模型适配器（默认选 `openai_chat_compatible` 即可）

**提示**：
- 可以配置多个 API 提供商（如同时配置 DeepSeek 和 Moonshot）
- 不同专家可以使用不同的 API，灵活切换

---

### 步骤 3/8：专家配置

这一步是定义你的专家小组。

**什么是专家？**
这里的"专家"实际上是 AI 模拟的不同角色。你需要为每个专家设置：
- **姓名/称呼**：如"张教授"、"李博士"
- **职位/角色**：如"计算机科学系主任"
- **机构类型**：高校、企业、政府机构、MCN 等
- **地区**：如北京、上海、杭州
- **专业领域**：如人工智能、市场营销、产业政策
- **LLM 提供商和模型**：选择步骤2中配置的 API

**操作步骤：**

1. 选择 **手动添加** 或 **AI 生成**
   - 手动添加：逐个输入专家信息
   - AI 生成：输入专家数量，让 AI 自动生成专家列表
2. 设置每个专家的 LLM 配置（选择使用哪个 API 和模型）
3. 设置每位专家的**打分倾向**：
   - 激进（倾向给出高分）
   - 保守（倾向给出低分）
   - 中立（客观打分）

**建议**：至少配置 3-4 位不同背景的专家，以获得多元视角。

---

### 步骤 4/8：轮次配置

这一步设置 Delphi 访谈的规则。

**Delphi 轮次说明**：

- **第一轮**：AI 专家根据自己的专业知识，对研究问题进行深入分析和解答
- **第二轮**：基于第一轮识别出的因素，AI 专家进行评分

**需要设置的内容**：

- **第一轮轮次**：每个专家进行几轮追问（建议 3-5 轮）
- **第二轮评分维度**：从哪些角度对因素进行评分
  - 例如：可行性、创新性、影响力、紧迫性
  - 每个维度可以设置权重（权重之和为 100%）
- **收敛阈值**：专家意见达到一致的标准（默认 0.5）
- **最大轮数**：最多进行几轮（建议 3 轮）

---

### 步骤 5/8：执行流程

这一步是自动运行的，程序会依次执行：

1. **Delphi 第一轮**：调用各位专家的 LLM 进行访谈
2. **因子编码**：从专家访谈中提取和汇总所有因素
3. **Delphi 第二轮**：调用专家对每个因素进行评分
4. **AHP 权重计算**：计算各因素的权重和一致性检验
5. **报告生成**：生成 Markdown 格式研究报告

**此步骤会自动执行，可能需要几分钟到十几分钟**，取决于专家数量和因素数量。

**屏幕会实时显示**：
- 当前正在调用哪位专家
- 当前正在执行哪个阶段
- 已完成进度

---

### 步骤 6/8：收敛检验与汇总

查看 Delphi 分析结果。

**收敛检验**是什么？
检验各位专家的意见是否趋于一致。如果变异系数（CV）≤ 0.5，说明专家意见已经收敛，可以进入下一轮或结束。

**显示内容**：
- 每个因素的均值、标准差、变异系数
- 专家意见是否收敛（收敛/未收敛）
- 专家观点汇总

---

### 步骤 7/8：AHP 权重分析

查看各因素的权重排序。

**AHP 是什么？**
层次分析法，通过两两比较判断各因素的重要性。

**显示内容**：
- 各因素权重（所有权重之和 = 100%）
- 一致性比率（CR）
- CR ≤ 0.1 表示通过一致性检验，结果可信

---

### 步骤 8/8：生成报告

查看和导出最终研究报告。

**报告内容包括**：
- 研究概述（标题、背景、目的）
- 专家小组介绍
- Delphi 访谈过程和结果
- 因子汇总
- AHP 权重分析结果
- 结论和建议

**导出格式**：
- Markdown（`.md`）— 可直接粘贴到 Word
- HTML — 可在浏览器中查看
- 交互式报告 — 支持更丰富的展示

---

## 输出文件说明

运行完成后，在 `run_result/` 文件夹下会生成以你的研究标题命名的文件夹，包含：

| 文件名 | 内容 |
|:---|:---|
| `project.json` | 你的项目配置信息 |
| `experts.json` | 专家配置信息 |
| `providers.json` | API 配置信息 |
| `interview_records.json` | 所有专家访谈记录 |
| `convergence_check.json` | 收敛检验结果 |
| `ahp_results.json` | AHP 权重计算结果 |
| `final_report.md` | 最终研究报告 |
| `analysis_tables.xlsx` | Excel 分析表格 |

## 提示和技巧

### 研究问题怎么写？
研究问题应该：
- 明确、具体
- 与你的研究主题高度相关
- 适合通过专家访谈来回答

**好的例子**：
- "直播带货场景中，消费者信任主播的核心要素有哪些？"
- "影响新能源汽车充电站选址的关键因素是什么？"

### 专家数量多少合适？
- 建议 **4-6 位**专家
- 太少：意见缺乏代表性
- 太多：分析复杂度增加

### 专家背景怎么选？
尽量选择**互补**的背景，例如：
- 高校教授（理论视角）
- 企业高管（实践视角）
- 政策研究员（政策视角）
- 行业从业者（一线视角）

### API 费用问题？
- Xi-API 按调用次数计费，学生通常每月几十元足够
- 选择价格低的模型（如 DeepSeek）可以大幅节省费用
- 不要让程序跑太多轮次，设置合理的最大轮数

## 故障排除

| 问题 | 解决方法 |
|:---|:---|
| 提示"API Key 无效" | 检查 Key 是否正确复制，是否有多余空格 |
| 程序中途卡住 | 按 Ctrl+C 重新运行，数据不会丢失 |
| 报告内容太少 | 增加研究问题数量，丰富研究背景描述 |
| 专家回答太相似 | 增加不同背景的专家，设置不同打分倾向 |
| 无法打开报告 | 使用 VS Code 或 Typora 等工具打开 Markdown 文件 |

## 技术支持

如遇问题，可尝试：
1. 查看终端/命令行中的错误提示
2. 使用 AI 编程助手（Claude Code、OpenClaw 等）帮你排查
3. 重新运行安装脚本
4. 检查 API Key 是否还有余额
